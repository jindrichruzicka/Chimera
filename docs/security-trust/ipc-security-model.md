# IPC Security Model & Trust Boundaries

> §9 of the Chimera architecture.
> Related: [Electron Shell & IPC Bridge](../core-components/electron-shell-ipc-bridge.md) · [Fog of War](fog-of-war-cryptographic-commitment.md) · [Architecture Invariants](../executive-architecture/architecture-invariants.md)

## Security Boundary Table

| Boundary             | Rule                                                                                                                                          |
| -------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| Renderer → Main      | Only through `contextBridge`. `nodeIntegration: false`. `contextIsolation: true`.                                                             |
| Client → Server      | All actions validated server-side. Client cannot mutate state by crafting a WebSocket message.                                                |
| Lobby tokens         | Short-lived random tokens issued by host on `JOIN`; prevents unauthenticated connections. No persistent auth required for peer-hosted play.   |
| IPC surface          | Preload exposes typed, enumerated methods only. No `eval`, no arbitrary Node.js access from renderer.                                         |
| Action checksums     | `ACTION` messages carry a CRC32 of `(playerId + tick + actionPayload)` to detect tampering or corruption.                                     |
| State obfuscation    | `GameSnapshot` never crosses any process boundary. `StateProjector` is the mandatory gate between simulation and all outbound messages.       |
| Commitment integrity | `CommitmentScheme.verify()` is called client-side on every `REVEAL` message before the value is trusted. Failures are surfaced to the player. |
| Host renderer trust  | Host's own renderer receives `PlayerSnapshot`, not `GameSnapshot`. Host player cannot gain info advantage via devtools inspection.            |

## §9.1 IPC Attack Surface Audit

The preload bridge (`electron/preload/api.ts`, §4.1) exposes `window.__chimera` with the namespaces below. This is the single entry point from the untrusted renderer into the trusted main process. **Every change to this surface must update this table.**

| Namespace                   | Writes / side-effects                                                                                                                                                                                                                                                                                                                                                                | Main-process validator                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    | Trust classification                                                                                                             | Gameplay authority?                                                                |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| `game`                      | `sendAction()` dispatches `EngineAction`s into `ActionPipeline`; `getCurrentSnapshot()` is read-only replay of the last projected `PlayerSnapshot`.                                                                                                                                                                                                                                  | `sendAction`: `parseInvokeRequest(EngineActionSchema)` at `ipcMain`; `ActionPipeline.validate()` then runs `ActionDefinition.validate()` + `parsePayload`. Unknown types rejected at `registry.resolve()`. `getCurrentSnapshot`: no payload, returns only `PlayerSnapshot / null` (inv. #95).                                                                                                                                                                                                                                                                                                                                                                                                                                             | Sanitiser-gated for actions; main-authoritative read-only for snapshot replay.                                                   | **Yes** for `sendAction()` only; `getCurrentSnapshot()` has no gameplay authority. |
| `lobby`                     | Create/join/close lobby; kick players (host only). `JOIN` may include `reconnectPlayerId` to reclaim a disconnected seat, or bounded saved-seat `claims` (≤16 opaque `{matchId, playerId}` entries, ids ≤64 chars) to reclaim a restored seat (F68/#821).                                                                                                                            | `parseInvokeRequest(HostLobbyParamsSchema / JoinLobbyParamsSchema)`. `LobbyManager` verifies caller is host for privileged ops. Reconnect reclaim (shared `resolveRestoredSeat`) requires an identity that actually connected this session, is not currently connected, was not forgotten by LEAVE, and is never the host id — merely-seeded restored seats are not reconnectable. Seat claims are wire-schema-bounded (`WIRE_MAX_JOIN_CLAIMS` / `WIRE_MAX_JOIN_CLAIM_ID_LENGTH`) and granted only when the claim's `matchId` equals the host's restored match and the playerId is a restored (non-host) seat not currently connected — anything else degrades to a fresh id, never a crash.                                              | Main-authoritative; reconnect/claim reclaim is token-gated/local-only risk acceptance (no persistent auth in peer-hosted model). | No (session control only).                                                         |
| `saves`                     | List / load / save / delete save files.                                                                                                                                                                                                                                                                                                                                              | `parseInvokeRequest(GameIdSchema / SaveRequestSchema / SlotIdSchema)`. | `profile`                   | Read local profile(s); update local profile.                                                                                                                                                                                                                                                                                                                                         | `ProfileManager` (local) + `ProfileSanitizer.admit()` for network attestation (inv. #61).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 | Sanitiser-gated for wire; main-authoritative for disk.                                                                           | No (cosmetic only, inv. #59–62).                                                   |
| `replay`                    | List / export / open / delete replay files. `export-current-match` accepts an optional `'save' \| 'view'` intent that gates the `onExported` push (`'view'` exports for a path only, no toast). Two one-way main → renderer push subscriptions: `onNavigate` (`chimera:replay:navigate`) and `onExported` (`chimera:replay:exported`), each carrying only a replay file-path string. | `ReplayManager` — path confined to `userData/replays/`; format-version + `(engineVersion, gameId, gameVersion)` checked on load (inv. #71). The `export-current-match` intent is validated by `ReplayExportIntentSchema` (`z.enum(['save','view']).catch('save')`) — a fail-safe default, so a malformed renderer payload can only ever trigger the cosmetic toast, never suppress it. Push subscriptions are receive-only (`ipcRenderer.on`); the renderer cannot invoke them and they carry no gameplay authority.                                                                                                                                                                                                                      | Main-authoritative.                                                                                                              | No (playback uses live `ActionPipeline`, inv. #70).                                |
| `chat`                      | Send chat; subscribe to inbound; local mute.                                                                                                                                                                                                                                                                                                                                         | `ChatRelay.relay()` — length cap, rate limit, scope check (inv. #73).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     | Sanitiser-gated.                                                                                                                 | No (never an `EngineAction`, inv. #72).                                            |
| `logs`                      | Emit log entry; read recent logs for export.                                                                                                                                                                                                                                                                                                                                         | `parseInvokeRequest(RendererLogEntrySchema)` — log-level whitelist; every string field the schema names is capped (`message` 4096, `source.module` 256, `error.name` 256, `error.message` 4096, `error.stack` 8192). `source.process` is absent from the schema, so a forged value is stripped and the handler substitutes the constant `'renderer'`; `timestamp` is overridden with `Date.now()`. `context` is bounded in **shape** (`z.record`) but deliberately **not in size** — it carries arbitrary structured diagnostics, so a renderer can enlarge a log entry, though not forge its origin. A failing entry is dropped, never truncated, so `rendererLogger.ts` truncates to the same caps before emitting. Never re-broadcast. | Renderer-write / main-persist. | `lobbyDiscovery` (optional) | LAN / Steam browse list.                                                                                                                                                                                                                                                                                                                                                             | Provider-specific; browse result is read-only.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            | Main-authoritative; read-only.                                                                                                   | No.                                                                                |

## Audit Procedure for New Namespaces

When adding or modifying a namespace, follow all six steps:

1. **Declare shape in `electron/preload/api.ts`** as a typed, enumerated method set. No generic proxies, no pass-through `invoke`.
2. **Add a row to the table above** covering all six columns.
3. **Register the main-process handler in `electron/main/ipc-handlers.ts`** with explicit input validation at the first line of each handler.
4. **If the surface accepts structured payload**: write a Zod (or equivalent) schema and reject before touching any manager.
5. **If the surface can influence gameplay**: confirm it goes through `ActionPipeline` — never through a direct manager mutation.
6. **Add at least one invariant in [Architecture Invariants](../executive-architecture/architecture-invariants.md)** capturing the trust rule and link from the namespace description.

## Relevant Invariants

| #       | Rule                                                                                         |
| ------- | -------------------------------------------------------------------------------------------- |
| #1      | `GameSnapshot` never leaves the host's main process.                                         |
| #3      | `GameSnapshot` never crosses any process or network boundary.                                |
| #4      | The renderer reads state; it never writes state directly.                                    |
| #5      | All IPC methods are declared in `ipc-handlers.ts` and exposed only through `preload/api.ts`. |
| #6      | Network messages are validated before they touch the simulation.                             |
| #27–#29 | Debug namespace is absent in production; `webContents.id` validated on every debug IPC call. |
| #95     | `getCurrentSnapshot()` is read-only and returns only `PlayerSnapshot / null`.                |
